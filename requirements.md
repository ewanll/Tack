# Tack · 桌面待办工具 — 需求文档

> 版本：v1.0 · 2026-06-30
> 技术栈：Python + PyQt6 + QWebEngineView

---

## 一、产品概述

Tack 是一个桌面悬浮待办工具，核心形态是一个**透明悬浮时钟**，双击打开**周历+待办**主窗口，数据保存在**本地 JSON 文件** `tack_todos.json` 中。用户可以在设置中选择任意文件夹存放数据文件，如果路径设在 Google Drive / OneDrive 的本地同步目录，即可实现多设备同步。

## 二、功能需求

### 2.1 悬浮时钟（Floating Widget）

| # | 需求 | 说明 |
|---|---|---|
| F1 | 显示实时时间 HH:mm:ss | 秒数实时更新 |
| F2 | 显示今日待办计数 | 格式：`周三 今日待办 3` |
| F3 | 完全透明背景 | 只显示文字，无背景框 |
| F4 | 文字始终可读 | 白色 + 多层深色文字阴影，适配任意桌面背景 |
| F5 | 鼠标拖拽移动位置 | 拖拽后自动记住位置 |
| F6 | 双击打开主窗口 | |
| F7 | 位置持久化 | 下次启动恢复上次位置 |

### 2.2 主窗口

| # | 需求 | 说明 |
|---|---|---|
| W1 | 居中弹出式窗口 | 带缩放动画，点击遮罩层或 Esc 关闭 |
| W2 | 周日历（周/月视图可选） | 显示一周或整月日期，有待办的日子显示小圆点 |
| W3 | 点击日历日期筛选待办 | 再次点击取消筛选 |
| W4 | 日期选择器 | 添加待办时可选择日期（默认今天/选中的日期） |
| W5 | 待办列表（CRUD） | 增删改查、勾选完成、双击编辑 |
| W6 | 待办日期标签 | 全部视图下每条待办显示日期芯片（今天/周末/过去 颜色区分） |
| W7 | 待办时间标签 | 显示创建时间（"3m前""2h前"等） |
| W8 | 搜索过滤 | 列表上方搜索框，实时过滤 |
| W9 | 完成待办折叠 | 已完成的默认收起，点击展开 |
| W10 | 拖拽排序 | 按住手柄拖拽调整待办顺序 |
| W11 | Ctrl+V 粘贴多行 | 一次粘贴多行文本拆成多条待办 |
| W12 | 导出本周待办 | 复制到剪贴板 |

### 2.3 设置

| # | 需求 | 说明 |
|---|---|---|
| S1 | 语言切换 | 中文简体 / English，全部界面文字切换 |
| S2 | 悬浮字体颜色 | 系统取色器，持久化 |
| S3 | 悬浮字号 | 滑块 1.5~5rem，实时生效，持久化 |
| S4 | 悬浮窗透明度 | 滑块 20%~100%，持久化 |
| S5 | 日历显示周数 | Toggle 开关 |
| S6 | 开机自启 | Toggle 开关，写入 Windows 注册表 |
| S7 | 数据文件路径 | 自定义文件夹存放 tack_todos.json |

### 2.4 数据同步方案

Tack 不直接与任何云盘 API 对接。数据文件 `tack_todos.json` 保存在用户指定的本地文件夹中。

**多设备同步方式：**
- 用户在设置中选择 Google Drive / OneDrive / Dropbox / iCloud 的本地同步目录
- 云盘的桌面客户端会自动把文件同步到云端
- 在另一台电脑上安装 Tack，指向同一个云盘目录即可
- 数据就是本地的 JSON 文件，完全离线可用

| # | 需求 | 说明 |
|---|---|---|
| D1 | 数据格式 | 单文件 tack_todos.json，含版本号和待办数组 |
| D2 | 默认路径 | `%APPDATA%/Tack/tack_todos.json` |
| D3 | 自定义路径 | 设置中通过文件夹选择器更改 |
| D4 | 自动创建 | 如果文件不存在，首次保存时自动创建 |
| D5 | 离线可用 | 完全离线，不需要网络 |

### 2.5 系统托盘

| # | 需求 | 说明 |
|---|---|---|
| T1 | 最小化到托盘 | 关闭主窗口后程序在后台运行 |
| T2 | 托盘菜单 | 打开/退出 |

## 三、非功能需求

| # | 需求 | 说明 |
|---|---|---|
| N1 | 跨平台 | Windows 优先 |
| N2 | 内存占用 | < 200MB 常驻内存 |
| N3 | 离线优先 | 完全离线，不需要网络 |
| N4 | 启动速度 | < 3 秒显示悬浮时钟 |
| N5 | 数据安全 | tack_todos.json 只包含待办文本，无敏感信息 |

## 四、数据模型

```json
{
  "version": 1,
  "lastSyncedAt": "2026-06-30T15:30:00.000Z",
  "todos": [
    {
      "id": 1,
      "text": "买牛奶",
      "done": true,
      "date": "2026-06-30",
      "createdAt": "2026-06-30T08:00:00.000Z",
      "sortOrder": 0
    }
  ]
}
```

## 五、项目架构

```
tack/
├── main.py                  # 入口
├── requirements.txt
├── app/
│   ├── __init__.py
│   ├── clock_widget.py      # 悬浮时钟窗口（纯 PyQt6）
│   ├── main_window.py       # 主窗口（QWebEngineView + 桥接）
│   ├── bridge.py            # JS↔Python QWebChannel 通信桥
│   └── storage.py           # 本地 JSON 文件读写
├── ui/
│   └── app.html             # 主窗口 HTML（含日历+待办+设置+语言切换）
└── data/                    # 运行时数据目录（由 storage.py           # 本地 JSON 文件读写
    ├── todos.json
    └── settings.json
```
